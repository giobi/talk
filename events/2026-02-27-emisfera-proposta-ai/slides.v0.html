Brain AI

Assistenti AI contestuali per Emisfera

Febbraio 2026 · Giobi Fasoli

Il problema

  • Colleghi che incollano dati sensibili in ChatGPT
  • Nomi, cognomi, ticket, dati clienti su server terzi
  • Viola GDPR e AI Act
  • Proibire non funziona — serve un'alternativa

L'AI la usano già. La domanda non è se, ma come.

Cos'è il Brain AI

  • Non è un prodotto — è una metodologia
  • Ogni utente ha un "cervello": file Markdown strutturati
  • L'AI sa chi sei, i tuoi progetti, i tuoi colleghi
  • File locali su server Emisfera, non in cloud

Portabile: se cambiamo modello AI domani, il Brain si sposta tale e quale.

Architettura

INPUT BRAIN (server locale) OUTPUT ───── ────────────────────── ────── Email ┐ boot/ identità ┌ Risposte chat Calendario ├──────► wiki/ persone, progetti ───► Bozze email Ticket │ diary/ storico sessioni │ Report, siti Database ┘ shared/ knowledge aziendale └ Automazioni ▲ │ └── LLM ──┘

Sicurezza

Cosa serve sapere per i sistemi

Modello di sicurezza

  • 1. File su server Emisfera, non su cloud esterni
  • 2. Criptazione Linux per cartelle private utente
  • 3. Admin di sistema non legge contenuti privati
  • 4. AI legge ma non scrive senza approvazione utente
  • 5. Ogni azione loggata — audit completo via Git

Cosa transita in rete

Resta in sede

  • File del Brain (wiki, diary)
  • Tool e integrazioni
  • Dati sensibili e credenziali
  • Cartella shared aziendale

Esce via API

  • Solo il contesto della chat attiva
  • Filtrabile e monitorabile
  • No training sui nostri dati (policy Anthropic/OpenAI)
  • Alternativa futura: GPU locale

Permessi e accessi

PrivatoSolo l'utente. Criptato. Email, diary, note personali.
CondivisoTeam/reparto. Prassi, protocolli, knowledge base.
PubblicoTutta l'azienda. Documentazione, FAQ, procedure.

L'utente decide cosa condividere. Il sistema non "aspira" dati automaticamente.

Perché non Docker?

  • L'isolamento è a livello OS: un utente Linux per brain
  • È lo stesso meccanismo che Docker usa sotto il cofano
  • Tool condivisi, dipendenze identiche, un server → zero overhead

L'architettura è predisposta per i container. Quando servirà scalare a multi-tenant, si aggiunge Docker sopra senza riscrivere nulla.

Docker per i servizi (WebSSH, proxy, bot) — filesystem Linux per i dati utente.
